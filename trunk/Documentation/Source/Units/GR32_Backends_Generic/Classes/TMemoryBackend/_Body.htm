Description

This backend allocates the memory required for the surface directly in RAM. On Windows / VCL it does not acquire any GDI handles or resources.

It does not support any of the interfaces required for TBitmap32, making it lightweight but also feature limited. Temporarily switching to a more sophisticated backend like TGDIBackend can alleviate this problem.

TMemoryBackend is the default backend of TCustomBitmap32.
